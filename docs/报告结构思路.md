# 《数字图像处理》期末综合报告 — 结构思路

## 课题定位
**课题方向**：医学图像处理
**课题名称**：基于ResNet50与混合专家(MoE)架构的眼底疾病智能诊断系统
**核心亮点**：双模型对比 + 集成诊断 + 图像预处理 + MATLAB风格量化分析 + PyQt5软件封装

---

## 报告整体思路（叙事主线）

以"眼底疾病智能诊断"为应用场景，构建一条完整的技术链条：
> **图像预处理（CLAHE增强）→ 双模型训练（ResNet50后融合 vs MoE门控融合）→ 对比实验与集成诊断 → MATLAB风格量化分析 → 软件封装展示**

这条主线覆盖了课程考查的所有基本点（预处理、识别、分类、量化分析），同时通过MoE架构、双模型集成、软件封装等技术亮点获取加分。

## 报告章节结构与内容规划

### 第一章 引言（约1.5-2页）

**1.1 研究背景与意义**
- 眼底疾病是全球主要致盲原因（糖尿病视网膜病变、青光眼、AMD等），早期筛查意义重大
- 传统人工阅片效率低、基层缺乏专业眼科医生
- 深度学习在医学影像分析中的应用价值：自动化、高效率、辅助诊断

**1.2 国内外研究现状**
- 简述深度学习在眼底图像分析中的发展：从CNN到Vision Transformer
- ResNet在医学图像分类中的广泛应用
- MoE架构在大模型时代的新趋势（参考你已有的MOE文档内容）
- 指出现有研究的不足：多数方法仅使用单模型、缺乏左右眼融合机制

**1.3 研究目标与创新点**
- 构建完整的眼底疾病智能诊断流程（预处理→识别→分析→展示）
- 创新点清单（4-5个）：
  1. 双模型对比框架：ResNet50后融合 vs MoE双流门控融合
  2. 智能集成诊断引擎：加权融合 + 共病分析 + 自适应阈值
  3. 多方法图像预处理量化评估体系（CLAHE/高斯/反锐化掩膜）
  4. MATLAB风格传统图像量化分析 + 深度学习分析的互补
  5. PyQt5桌面软件封装，完整的临床诊断交互界面

### 第二章 方法设计（约2-3页）

**2.1 系统总体架构**
- 画一张系统总框图：输入(左右眼眼底图)→图像预处理→ResNet50推理→MoE推理→集成诊断引擎→MATLAB量化分析→诊断报告
- 简要说明各模块的功能和相互关系

**2.2 图像预处理方法**
- 2.2.1 CLAHE（限制对比度自适应直方图均衡化）：LAB色彩空间，仅对L通道处理
- 2.2.2 高斯滤波去噪：5×5核，保持边缘
- 2.2.3 反锐化掩膜：边缘增强
- 2.2.4 完整预处理流水线：CLAHE → 高斯滤波 → 反锐化掩膜
- 每种方法的数学原理简述 + 为什么适合眼底图像

**2.3 ResNet50 模型设计**
- 2.3.1 残差学习原理（恒等映射，解决网络退化）
- 2.3.2 网络结构：ResNet50骨干 + 8类多标签分类头
- 2.3.3 后融合策略：左右眼独立推理 → Sigmoid概率 max 融合
- 2.3.4 迁移学习策略：ImageNet预训练 + 渐进式解冻微调

**2.4 混合专家（MoE）模型设计**
- 2.4.1 MoE架构原理：稀疏激活、动态路由
- 2.4.2 模型结构：
  - 共享ResNet50特征提取器（冻结参数）
  - 左右眼特征拼接（2048×2=4096维）
  - 门控网络（4096→512→8，Softmax输出专家权重）
  - 8个专家子网络（各3层MLP：4096→1024→512→8，含BN和Dropout）
  - 加权输出：y = Σ g_i · E_i(x)
- 2.4.3 Focal Loss训练：解决类别不平衡问题（α=0.25, γ=2.0）
- 2.4.4 与ResNet50的对比设计意图：特征级融合 vs 后融合

**2.5 集成诊断引擎**
- 2.5.1 加权融合：基于各模型在验证集上的逐类F1分数分配权重
- 2.5.2 动态权重调整：模型分歧度大时自动增强置信模型的权重
- 2.5.3 共病分析矩阵：基于医学先验知识（如糖尿病-高血压共现增强）
- 2.5.4 自适应阈值策略：各类别使用不同阈值（0.12-0.90区间）
- 2.5.5 置信度分级：高/中/低/需复核四级

### 第三章 实验与分析（约3-4页）

**3.1 数据集介绍**
- 数据集：Kaggle ODIR-5K（公开数据集）
- 规模：约3500患者、7000张眼底图像
- 8类标签：正常/糖尿病/青光眼/白内障/AMD/高血压/近视/其他异常
- 多标签特性：同一患者可能存在多种疾病
- 数据划分：按患者级别划分（防止左右眼数据泄漏），训练集/验证集/测试集
- 数据预处理：黑边裁剪、缩放至224×224、ImageNet标准化
- 数据增强：随机水平翻转、旋转(±30°)、颜色抖动
- 类别平衡：过采样少数类至多数类数量

**3.2 实验环境与超参数**
- 硬件：GPU型号/CPU、显存
- 软件：PyTorch、OpenCV、PyQt5等
- 关键超参数表格（两个模型的对比）：

| 超参数 | ResNet50 | MoE |
|--------|----------|-----|
| 优化器 | SGD + Momentum(0.9) | AdamW |
| 学习率 | 0.001 | CosineAnnealingWarmRestarts |
| Batch Size | 8 | 8 |
| Epoch | 50 | 基于早停 |
| 损失函数 | BCEWithLogitsLoss | Focal Loss (γ=2.0) |
| 正则化 | Weight Decay 1e-4 | Gradient Clipping + Dropout |

**3.3 训练过程与结果**
- 训练曲线图（Loss曲线 + Accuracy曲线，ResNet50和MoE对比）
- ResNet50最佳结果：Epoch 48, Val Accuracy 91.91%, Test Accuracy 88.78%
- MoE训练结果（附对应指标）
- 测试集分类报告表格（8类的Precision/Recall/F1-Score）
- 分析：哪些类别表现好（近视、青光眼F1>0.95），哪些类别有挑战（正常、糖尿病F1偏低）

**3.4 图像预处理对比实验**
- 三种预处理方法的视觉对比图（原图 vs CLAHE vs 高斯 vs 完整流水线）
- 量化指标对比表（PSNR、SSIM、CII、Entropy、EME）
- CLAHE增强后的特征改善效果（直方图对比、Sobel梯度对比）
- 分析：CLAHE作为推荐预处理方法的原因（EME增益约2倍）

**3.5 ResNet50 vs MoE 对比实验**
- 综合指标对比表（Accuracy, Precision, Recall, F1, Hamming Loss, AUC）
- 逐类F1对比柱状图
- ROC曲线对比
- 推理速度对比（FPS/单张图像）
- 模型参数量对比
- 讨论：两种模型的优劣势——
  - ResNet50优势：轻量、稳定、正常/糖尿病/AMD/其他异常表现好
  - MoE优势：青光眼/白内障/近视表现好（特征级融合更敏感）

**3.6 集成诊断引擎效果**
- 单模型 vs 集成模型性能对比表
- 消融实验结果（逐一去掉加权融合/共病分析/自适应阈值，观察性能变化）

### 第五章 结论与展望（约0.5-1页）

**5.1 工作总结**
- 完成了眼底疾病智能诊断系统的完整设计：预处理→双模型训练→集成诊断→量化分析→软件封装
- ResNet50达到88.78%测试准确率，集成引擎进一步提升诊断可靠性
- 完整预处理流水线（CLAHE+高斯+反锐化掩膜）有效增强眼底图像质量
- PyQt5桌面软件实现了友好的人机交互

**5.2 不足与改进方向**
- 数据局限：ODIR-5K数据集规模有限、标签可能存在噪声
- 模型局限：缺少分割模块（血管分割/病灶定位可增加可解释性）
- 临床局限：尚未在真实临床环境中验证
- 展望：
  - 引入病灶分割模块（U-Net/Mask R-CNN）实现更精准的病灶定位
  - 探索多模态融合（OCT影像+眼底彩照）
  - 生成式AI辅助报告撰写（生成诊断描述文本）
  - 边缘端部署（模型量化、轻量化）支持基层筛查

## 写作注意事项

1. **格式规范**：严格按照课程要求（正文12号字、标题14号字、行间距固定值20磅、图表居中并附标题）
2. **学术规范**：所有图表必须有编号和标题（如图1、表2）；参考文献按规范格式引用
3. **量化分析**：所有结论必须有数据支撑，避免"效果好"这种模糊表述，应该说"准确率提升X%"
4. **杜绝AI痕迹**：课程明确说了"明显的AI生成报告内容"会不及格，注意用自己的语言重新组织，多一些具体的实验细节和个人思考
5. **突出工作量**：展示项目的完整性和丰富度——双模型、对比实验、预处理体系、GUI封装、量化分析
6. **图多字精**：8-12页的篇幅限制，用图表高效传达信息，文字简洁有力
